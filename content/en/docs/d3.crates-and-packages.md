## Crate

- A crate is the smallest unit of compilation in Rust.
- A crate compiles as a single unit. If it contains modules in separate files, the compiler builds an isolated module tree starting from the crate root before compilation.
- Each crate maintains its own module tree rooted at the crate root:
  - Binary crates: `src/main.rs` or a `.rs` file in the `src/bin/` directory
  - Library crates: Usually `src/lib.rs`
- Main crate types are,
  | Type         | Description                                                 |
  | ------------ | ------------------------------------------------------------|
  | `bin`        | Executable program                                          |
  | `lib`        | Let the compiler decide the library type. Defaults to `rlib`|
  | `rlib`       | Static library for linking into other Rust crates.          |
  | `staticlib`  | Static library for linking into non-Rust programs           |
  | `dylib`      | Rust dynamic library (rarely used)                          |
  | `cdylib`     | Dynamic library for C/FFI and other non-Rust programs       |
  | `proc-macro` | Library crate that defines procedural macros                |

  > [!tip]
  > To configure the library crate type/ types, we can use the `[lib]` section of `Cargo.toml`.
  > ```toml
  > [lib]
  > crate-type = ["rlib", "cdylib"] // Rust library + C-compatible dynamic library
  > ```

## Package

- A package is a Cargo project containing one or more crates, described by a `Cargo.toml` manifest.
- A Cargo package can contain:
  - Zero or more binary crates
  - Zero or one library crate
- Typical package layouts are,
  - `src/main.rs`: A package with single binary crate.
  - `src/lib.rs`: A package with single library crate.
  - `src/bin/*.rs`: A package with multiple binary crates.
  - `src/main.rs` + `src/lib.rs`: A package with 1 binary crate + 1 library crate.
  - `src/bin/*.rs` + `src/lib.rs`: A package with multiple binary crates + 1 library crate.

## Common Package Layouts

### Single Binary

```shell {title="Folder Structure"}
my_app
├── Cargo.toml
└── src
    └── main.rs
```

> [!lab]
> ```shell
> cargo new my_app
> cd my_app && cargo run
> ```

```shell {title="Folder Structure"}

    └── lib.rs
```

```shell {title="Folder Structure"}
my_app
├── Cargo.toml
└── src
    └── bin
        ├── app.rs
        └── cli.rs
```

{{< tabs count=3 label="Select File" >}}
{{< tab label="app.rs" >}}
```rust {title="src/bin/app.rs"}

    println!("Hello App!");
}
```
{{< /tab >}}
{{< tab label="cli.rs" >}}
```rust {title="src/bin/cli.rs"}

    println!("Hello Cli!");
}
```
{{< /tab >}}
{{< tab label="Cargo.toml" >}}
```toml {title="Cargo.toml"}
[package]
name = "my_app"
version = "0.1.0"
edition = "2024"
default-run = "app"

[[bin]]
name = "app"
path = "src/bin/app.rs"

[[bin]]
name = "cli"
path = "src/bin/cli.rs"

[dependencies]
```
{{< /tab >}}
{{< /tabs >}}

> [!lab]
> ```shell
> cargo new my_app && cd my_app
> mkdir src/bin && mv src/main.rs src/bin/app.rs && cp src/bin/app.rs src/bin/cli.rs
> $copy above file content
> ```
> - Run `app` binary crate: `cargo run --bin app` / `cargo run` (as `Cargo.toml` -> `default-run = "app"`)
> - Run `cli` binary crate: `cargo run --bin cli`

## Adding Dependencies

- A project dependency can be added from:
  - [crates.io](https://crates.io) - Rust's official package registry
  - A Git repository
  - A folder path

> [!tip] Tip: `cargo add <package>`
> - `-n` / `--dry-run`: Preview changes without modifying `Cargo.toml`. Ex: `cargo add uuid -n`
> - `-F` / `--features <features>`: Enable specific features. Ex: `cargo add uuid -F v7`
>   - `uuid = { version = "1.23.4", features = ["v7"] }` (💡Cargo.toml [dependencies])
> - `--rename <name>`: Rename the dependency. Ex. `cargo add uuid --rename id`
>   - `id = { version = "1.23.4", package = "uuid" }` (💡Cargo.toml [dependencies])
> - Source: A Git repository
>   - `--git <URI>`: Git repository location.
>   - `--branch <BRANCH>`: Git branch to download the crate from
>   - `--tag <TAG>`: Git tag to download the crate from.
>   - `--rev <REV>`: Git reference to download the crate from
>   - Ex. 
>     - `cargo add uuid --git https://github.com/uuid-rs/uuid`
>     - `cargo add uuid --git https://github.com/uuid-rs/uuid --tag v1.23.4`
> - Source: A local crate
>   - `--path <PATH>`: Filesystem path to local crate to add
>   - Ex. `cargo add foo --path foo` (💡`my_app/foo/`)
