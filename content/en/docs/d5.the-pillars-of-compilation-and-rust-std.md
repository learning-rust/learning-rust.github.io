## `rustc`, `proc_macro`, `core`, `alloc` & `std`

![d5-rustc-core-alloc-and-std.png](/docs/d5-rustc-core-alloc-and-std.jpg)

Rust language elements are defined and implemented across many major layers.

- `rustc`:
  - The Rust compiler + defines and implement the language
- `proc_macro`:
  - Compile-time API to implement procedural macros (derive, attribute, and function-like macros)
  - Generates and transform Rust code at compile-time
- `core`:
  - Dependency-free foundation(Not depend on an OS or an allocator)
  - Ex. Primitive types, traits, iterators, Option, Result, macros
- `alloc`:
  - Depends on a global allocator, but not on OS services
  - Ex. Vec, String, Box, Rc, Arc, collections, allocator APIs
- `std`:
  - The Rust standard library
  - Depends on operating system services or a supported platform runtime
  - Ex. Files, networking, threads, processes, time, environment

> [!tip]
> - [`rustc`](https://rustc-dev-guide.rust-lang.org) is the official compiler and core executable of the Rust language toolchain. It transforms Rust source code into optimized machine binaries by [parsing syntax](https://rustc-dev-guide.rust-lang.org/syntax-intro.html), lowering code into [internal representations](https://rustc-dev-guide.rust-lang.org/hir.html), checking types, resolving traits, enforcing ownership and lifetime rules, and performing borrow checking. It uses pluggable code generation backends to produce binaries for different targets and works with libraries like core that provide compiler-recognized language items and fundamental Rust definitions.
> 
> - [`proc_macro`](https://doc.rust-lang.org/proc_macro/) is a built-in compiler library that provides infrastructure for procedural macros and compile-time code generation. It allows developers to create custom derive macros, attribute macros, and function-like macros that operate on Rust syntax as token streams, transforming and generating code during compilation before final type checking.
>
> - [`core`](https://doc.rust-lang.org/core/index.html) is a minimal, dependency-free foundation needed by almost all Rust programs, including `#![no_std]` environments. It contains [primitive types](https://doc.rust-lang.org/core/all.html#primitives), [constants](https://doc.rust-lang.org/core/all.html#constants), [functions](https://doc.rust-lang.org/core/all.html#functions), [structs](https://doc.rust-lang.org/core/all.html#structs), [enums](https://doc.rust-lang.org/core/all.html#enums), [traits](https://doc.rust-lang.org/core/all.html#traits), [macros](https://doc.rust-lang.org/core/all.html#macros), [attribute macros](https://doc.rust-lang.org/core/all.html#attributes) and [derive macros](https://doc.rust-lang.org/core/all.html#derives). And it has no knowledge about heap allocations, operating system services, higher-level libraries and `libc`.
>
> - [`alloc`](https://doc.rust-lang.org/alloc/all.html) builds on top of `core` and provides heap-allocated data structures. It requires a global allocator but not an operating system. It contains its own [structs](https://doc.rust-lang.org/alloc/all.html#structs), [enums](https://doc.rust-lang.org/alloc/all.html#enums), [traits](https://doc.rust-lang.org/alloc/all.html#traits), [functions](https://doc.rust-lang.org/alloc/all.html#functions), [macros](https://doc.rust-lang.org/alloc/all.html#macros) needed to allocate, own, and manage heap-allocated values.
>
> - [`std`](https://doc.rust-lang.org/std/index.html) builds on top of both `core` and `alloc` and provides higher-level, operating system–integrated functionality such as I/O, files, networking, processes, threads time, environment access, and other facilities to support general-purpose Rust applications. It also re-exports commonly used items originally defined in `core` and `alloc` through `std` making them easier to remember and regroup with shorter paths. So, developers rarely need to remember the original paths. Additionally, `std` automatically imports commonly used items into every Rust module via the [`std prelude`](https://doc.rust-lang.org/std/prelude/index.html), except in `#![no_std]` programs, which use the [`core prelude`](https://doc.rust-lang.org/core/prelude/index.html) instead.
