The Pillars of Compilation and Rust STD

rustc, proc_macro, core, alloc & std

Rust language elements are defined and implemented across many major layers.

  • rustc:
    • The Rust compiler + defines and implement the language
  • proc_macro:
    • Compile-time API to implement procedural macros (derive, attribute, and function-like macros)
    • Generates and transform Rust code at compile-time
  • core:
    • Dependency-free foundation(Not depend on an OS or an allocator)
    • Ex. Primitive types, traits, iterators, Option, Result, macros
  • alloc:
    • Depends on a global allocator, but not on OS services
    • Ex. Vec, String, Box, Rc, Arc, collections, allocator APIs
  • std:
    • The Rust standard library
    • Depends on operating system services or a supported platform runtime
    • Ex. Files, networking, threads, processes, time, environment
Tip

  • rustc is the official compiler and core executable of the Rust language toolchain. It transforms Rust source code into optimized machine binaries by parsing syntax, lowering code into internal representations, checking types, resolving traits, enforcing ownership and lifetime rules, and performing borrow checking. It uses pluggable code generation backends to produce binaries for different targets and works with libraries like core that provide compiler-recognized language items and fundamental Rust definitions.

  • proc_macro is a built-in compiler library that provides infrastructure for procedural macros and compile-time code generation. It allows developers to create custom derive macros, attribute macros, and function-like macros that operate on Rust syntax as token streams, transforming and generating code during compilation before final type checking.

  • core is a minimal, dependency-free foundation needed by almost all Rust programs, including #![no_std] environments. It contains primitive types, constants, functions, structs, enums, traits, macros, attribute macros and derive macros. And it has no knowledge about heap allocations, operating system services, higher-level libraries and libc.

  • alloc builds on top of core and provides heap-allocated data structures. It requires a global allocator but not an operating system. It contains its own structs, enums, traits, functions, macros needed to allocate, own, and manage heap-allocated values.

  • std builds on top of both core and alloc and provides higher-level, operating system–integrated functionality such as I/O, files, networking, processes, threads time, environment access, and other facilities to support general-purpose Rust applications. It also re-exports commonly used items originally defined in core and alloc through std making them easier to remember and regroup with shorter paths. So, developers rarely need to remember the original paths. Additionally, std automatically imports commonly used items into every Rust module via the std prelude, except in #![no_std] programs, which use the core prelude instead.
